21  /**
22   * Base type for all field types.
23   * Typically every driver provides an implementation of this interface in order
24   * to be able to describe the fields of a resource. As this is completely tied to
25   * the implemented protocol, this base interface makes absolutely no assumption to
26   * any information it should provide.
27   *
28   * In order to stay platform and protocol independent every driver connection implementation
29   * provides a prepareField(String) method that is able to parse a string representation of
30   * a resource into it's individual field type. Manually constructing PlcField objects
31   * manually makes the solution less independent from the protocol, but might be faster.
32   */
